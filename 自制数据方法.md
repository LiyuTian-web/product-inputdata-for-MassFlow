# 自制 MassFlow 输入数据方法（以白格滑坡为例）

手册白格案例的位置是 **98°42′17.98″E，31°4′56.41″N**，也就是大约 **98.705°E，31.082°N**；手册中白格案例也是先获取坡体表面文件 `s.txt`，再圈定滑坡范围文件 `m.txt`，后续生成滑面和滑体厚度文件。 

## 一、下载 Copernicus DEM GLO-30

Copernicus DEM GLO-30 是 DSM，不是裸地 DTM，它代表地表反射面，包括植被、建筑等；它来自编辑后的 WorldDEM，做过水体平坦化、河流连续性、岸线和异常地形结构等处理。 数据由 TanDEM-X 任务在 2011—2015 年获取，对白格 2018 年滑坡来说，大体可以作为滑前地形使用。

### 推荐下载方式：OpenTopography

这个方式最简单，可以直接按经纬度范围裁剪下载，不用自己拼 1°×1° 瓦片。OpenTopography 提供 Copernicus GLO-30 数据集，而且 2024 年已经更新到 ESA Copernicus GLO-30/GLO-90 的 DGED 2023_1 版本。

操作步骤：

1. 打开 OpenTopography，搜索 **Copernicus GLO-30 Digital Elevation Model**。
2. 进入数据页面后，选择 **Select a region** 或类似的框选下载方式。
3. 输入白格滑坡附近范围。你可以先用这个范围：

```
West  = 98.675
East  = 98.735
South = 31.055
North = 31.110
```

这个范围大约 5–6 km 见方，能覆盖源区、运动路径和金沙江堆积区，30 m 分辨率下网格量也不会太大。

1. 输出格式选择：

```
GeoTIFF
```

1. 下载后得到一个 `.tif` 文件，比如：

```
baige_cop30_raw.tif
```

### 备用方式：Copernicus Data Space Browser

Copernicus Browser 也可以浏览和下载 Copernicus 数据，Copernicus Data Space 官方说明它支持在线浏览、分析和下载数据。 如果你用这个方式，注意选择：

```
Data Collections: Copernicus DEM
Product: CopDEM GLO-30
```

检索时间最好往前拉到 2001 年，因为官方说明为了确保找到所有可用 Copernicus DEM 数据，建议把搜索时间范围扩展到 2001 年。

## 二、把 DEM 投影到 UTM 47N

下载的 Copernicus DEM 通常是经纬度坐标，不能直接用于 MassFlow 正式建模。因为 MassFlow 的距离、速度、摩擦参数都默认按米制理解，所以你必须先转投影。

白格滑坡经度约 98.7°E，建议统一成：

```
WGS_1984_UTM_Zone_47N
EPSG: 32647
单位：meter
```

ArcMap 操作：

```
ArcToolbox
→ Data Management Tools
→ Projections and Transformations
→ Raster
→ Project Raster
```

参数建议：

```
Input Raster: baige_cop30_raw.tif
Output Raster: baige_cop30_utm47.tif
Output Coordinate System: WGS_1984_UTM_Zone_47N
Resampling Technique: BILINEAR
Output Cell Size: 30
```

注意：这里用的是 **Project Raster**，不是 **Define Projection**。
 `Define Projection` 只是告诉 ArcGIS “它原来是什么坐标系”；`Project Raster` 才是真正把经纬度栅格转换成米制 UTM 栅格。

## 三、裁剪计算域

如果你下载时已经按白格范围裁剪过，这一步可以只是精修。如果下载的是较大范围，建议再裁剪一次。

ArcMap 操作：

```
ArcToolbox
→ Data Management Tools
→ Raster
→ Raster Processing
→ Clip
```

输入：

```
Input Raster: baige_cop30_utm47.tif
Output Raster: baige_cop30_utm47_clip.tif
```

裁剪范围要包含：

```
滑坡源区
滑坡运动路径
金沙江河道堆积区
两侧适当缓冲区
```

先不要裁得太小。你做第一次模拟时，宁可多留一点空间。

## 四、制作 s.txt

`s.txt` 就是 MassFlow 需要的滑前表面高程文件。你可以直接由裁剪后的 DEM 导出 ASCII。

ArcMap 操作：

```
ArcToolbox
→ Conversion Tools
→ From Raster
→ Raster to ASCII
```

参数：

```
Input Raster: baige_cop30_utm47_clip.tif
Output ASCII: s.asc
```

然后把文件名改成：

```
s.txt
```

打开 `s.txt`，前几行应该类似：

```
ncols         190
nrows         205
xllcorner     468xxx.xxx
yllcorner     3438xxx.xxx
cellsize      30
NODATA_value  -9999
```

只要 `xllcorner`、`yllcorner` 是几十万、三百多万这种米制坐标，就说明投影基本对了。不要出现经纬度小数，也不要出现软件案例里那种负数局部坐标。

## 五、制作 m.txt

`m.txt` 是物源区范围文件。你可以理解成：

```
物源区内 = 1
物源区外 = 0
```

这里一定要注意：**m.txt 画的是初始失稳物源区，不是最终堆积区，也不是整个受灾范围。**

### 1. 准备辅助图层

在 ArcMap 里加载：

```
baige_cop30_utm47_clip.tif
遥感影像底图
Hillshade 山体阴影
Slope 坡度图
```

生成山体阴影：

```
ArcToolbox
→ Spatial Analyst Tools
→ Surface
→ Hillshade
```

生成坡度：

```
ArcToolbox
→ Spatial Analyst Tools
→ Surface
→ Slope
```

### 2. 新建物源区面文件

在 ArcCatalog 里新建 shapefile：

```
Name: baige_source_polygon.shp
Feature Type: Polygon
Coordinate System: WGS_1984_UTM_Zone_47N
```

开始编辑，沿滑坡后缘、侧缘和源区边界人工勾画。白格滑坡源区可以参考滑坡后缘陡坎、裸露破坏区、坡面形态突变位置，以及滑前/滑后影像差异。

### 3. 添加字段

给 `baige_source_polygon.shp` 新建一个字段：

```
Field Name: mask
Type: Short Integer
Value: 1
```

也就是让这个物源区面内部值为 1。

### 4. 面转栅格

ArcMap 操作：

```
ArcToolbox
→ Conversion Tools
→ To Raster
→ Polygon to Raster
```

参数：

```
Input Features: baige_source_polygon.shp
Value field: mask
Output Raster: source_mask_raw.tif
Cellsize: 30
```

最关键的是点开 **Environments**，必须这样设置：

```
Processing Extent: Same as baige_cop30_utm47_clip.tif
Snap Raster: baige_cop30_utm47_clip.tif
Cell Size: Same as baige_cop30_utm47_clip.tif
Output Coordinate System: Same as baige_cop30_utm47_clip.tif
```

这一步决定 `m.txt` 能不能和 `s.txt` 完全对齐。

### 5. 把物源区外 NoData 改成 0

Polygon to Raster 后，物源区外通常是 NoData。你要把它变成 0。

ArcMap 操作：

```
Spatial Analyst Tools
→ Map Algebra
→ Raster Calculator
```

表达式：

```
Con(IsNull("source_mask_raw"), 0, 1)
```

输出：

```
source_mask_01.tif
```

### 6. 导出 m.txt

```
ArcToolbox
→ Conversion Tools
→ From Raster
→ Raster to ASCII
```

参数：

```
Input Raster: source_mask_01.tif
Output ASCII: m.asc
```

然后改名：

```
m.txt
```

## 六、检查 s.txt 和 m.txt 是否完全匹配

打开 `s.txt` 和 `m.txt`，前 6 行必须基本一致：

```
ncols
nrows
xllcorner
yllcorner
cellsize
NODATA_value
```

至少这几项必须一致：

```
ncols
nrows
xllcorner
yllcorner
cellsize
```

`s.txt` 后面的数据是高程值，`m.txt` 后面的数据应该主要是 0 和 1。

如果 `ncols`、`nrows` 或 `xllcorner`、`yllcorner` 不一样，说明你在做 `m.txt` 栅格化时没有设置：

```
Snap Raster
Processing Extent
Cell Size
```

需要重新做 `Polygon to Raster`。
